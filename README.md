# LocalNodeIndexer

## Бэкенд-индексатор событий сети Ethereum на Node.js + PostgreSQL

Прослушивает события смарт-контракта, обрабатывает их и сохраняет в БД. 
Стек: TypeScript, Viem, Drizzle ORM.

Задание:
Разработаnm сервис на Node.js/TypeScript, который подключается к Ethereum и слушает события уже существующего контракта Сервис должен:

    Подключаться к ноде через RPC (используйте Alchemy/Infura).

    Использовать Viem для работы с блокчейном.

    При появлении новых событий сохранять их в PostgreSQL 

    Обрабатывать реорганизацию цепочки (reorg) — через хранение blockNumber и при обнаружении более глубокой цепочки удалять неподтверждённые события.

    Использовать REST API на Hono (или Express) для получения списка депозитов пользователя.

    Обеспечить надёжность: при падении сервиса он должен продолжить с того же места (сохранять last processed block).

    Контейнеризировать сервис и БД с помощью Docker (docker-compose).

Технические требования:

    TypeScript.

    Drizzle ORM (или TypeORM) для работы с PostgreSQL.

    Линтер (ESLint), форматтер (Prettier).

    Покрытие юнит-тестами ключевых модулей (Jest).

    Результат: Сервис, который можно запустить локально, и который в реальном времени индексирует события.
